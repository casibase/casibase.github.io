ストレージプロバイダーの追加

このドキュメントは初心者向けに設計されたステップバイステップのチュートリアルです。 強力なナレッジベースシステムであるCasibaseとストレージプロバイダを統合するプロセスについて説明します。

はじめに​

Casibaseにストレージプロバイダを追加すると、データを効率的に管理および保存することができ、ナレッジベースシステムに不可欠なコンポーネントになります。

ストレージプロバイダーの統合が初めての場合でも、心配いりません。 誰でも実行できる簡単なステップにプロセスを分割しました。

ステップ 1: Casdoor と Casibase を展開する​

まだ実行していない場合は、Deploy Casdoor and Casibaseチュートリアルを参照してください。

ステップ2：新しいストレージプロバイダーの追加​

ストレージプロバイダーはデータを保存するために使用されます。 ホームページのIdentity - Providersボタンをクリックすることで、Casdoorに追加できます。

ステップ2.1：ストレージプロバイダーの追加​

Addボタンをクリックしてストレージプロバイダーを追加します。

ステップ2.2：ストレージプロバイダー情報の入力​

ストレージプロバイダー情報を入力し、Save & Exitボタンをクリックします。

ヒント

Casdoorは以下のような様々なストレージプロバイダーをサポートしています：

  • AWS S3
  • Azure Blob
  • Google Cloud Storage
  • MinIO
  • Qiniu Cloud Object Storage
  • Alibaba Cloud OSS ...

例

Alibaba Cloud OSSストレージプロバイダーの追加​
警告
  • Client ID：Alibaba Cloud OSSアカウントのAccessKey IDです。
  • Client Secret：Alibaba Cloud OSSアカウントのAccessKey Secretです。

****はAlibaba Cloud OSSアカウント情報のプレースホルダーです。

ステップ2.3：ストレージプロバイダーの表示​

ストレージプロバイダーを追加した後、ストレージプロバイダー情報を表示できます。

ヒント

ストレージプロバイダーはCasdoorから提供されています。 Casdoorにストレージプロバイダを追加し、Casibaseに追加できます。

詳細については、ステップ2：新しいストレージプロバイダーの追加を参照してください。
