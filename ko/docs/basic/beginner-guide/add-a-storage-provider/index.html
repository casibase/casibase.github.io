스토리지 제공자 추가하기

이 문서는 초보자를 위한 단계별 튜토리얼입니다. Casibase(우리의 강력한 지식 시스템)에 스토리지 제공자를 통합하는 과정을 안내합니다.

소개​

Casibase에 스토리지 제공자를 추가하면 데이터를 효율적으로 관리하고 저장할 수 있으며, 이는 지식 시스템의 중요한 구성 요소입니다.

처음으로 스토리지 제공자를 통합하는 경우라도 걱정하지 마세요. 우리는 누구나 따라할 수 있도록 과정을 간단한 단계로 나누었습니다.

단계 1: Casdoor와 Casibase 배포하기​

아직 배포하지 않았다면 Casdoor와 Casibase 배포하기 튜토리얼을 참조하세요.

단계 2: 새 스토리지 제공자 추가하기​

스토리지 제공자는 데이터를 저장하는 데 사용됩니다. 메인 페이지에서 Identity - Providers 버튼을 클릭하여 Casdoor에 추가할 수 있습니다.

단계 2.1: 스토리지 제공자 추가하기​

Add 버튼을 클릭하여 스토리지 제공자를 추가합니다.

단계 2.2: 스토리지 제공자 정보 입력하기​

스토리지 제공자 정보를 입력하고 Save & Exit 버튼을 클릭합니다.

팁

Casdoor는 다음과 같은 다양한 스토리지 제공자를 지원합니다:

  • AWS S3
  • Azure Blob
  • Google Cloud Storage
  • MinIO
  • Qiniu Cloud Object Storage
  • Alibaba Cloud OSS ...

예시

Alibaba Cloud OSS 스토리지 제공자 추가하기​
주의
  • Client ID: Alibaba Cloud OSS 계정의 AccessKey ID입니다.
  • Client Secret: Alibaba Cloud OSS 계정의 AccessKey Secret입니다.

****는 Alibaba Cloud OSS 계정 정보의 자리 표시자입니다.

단계 2.3: 스토리지 제공자 보기​

스토리지 제공자를 추가한 후 스토리지 제공자 정보를 볼 수 있습니다.

팁

스토리지 제공자는 Casdoor에서 제공됩니다. Casdoor에서 스토리지 제공자를 추가한 다음 Casibase에 추가할 수 있습니다.

자세한 내용은 단계 2: 새 스토리지 제공자 추가하기를 참조하세요.
